# 📸 UGC Telegram Bot — Poly Content Bot

## 🎯 Цель проекта

Создать рабочего Telegram-бота, который собирает фотографии и короткие истории от студентов и сотрудников, сохраняет их на **Яндекс.Диск** и в **Google Sheets** для дальнейшего использования в соцсетях, подкастах и презентациях.

---

## 🖥️ Инструкция по работе с сервером Beget (VPS)

### 1. Как зайти на сервер

1. Зайдите в личный кабинет: https://beget.com
2. Перейдите в раздел **«Облако» → VPS** (или «VPS / VDS»)
3. Найдите свой сервер и нажмите **«Управление»**
4. Нажмите кнопку **«Терминал»** (веб-консоль) — откроется терминал
5. Вы автоматически будете под пользователем `root`

> **Совет по копированию в веб-терминале Beget:**
>
> * Выделите текст → правая кнопка мыши → **«Вставить»**
> * Или используйте `Shift + Insert`

## ⚙️ Основные команды для управления ботом

```bash
# Перейти в папку проекта
cd /home/content-bot

# Проверить статус бота
systemctl status content-bot

# Перезапустить бота
systemctl restart content-bot

# Остановить бота
systemctl stop content-bot

# Посмотреть последние логи
journalctl -u content-bot -n 50 --no-pager

# Логи в реальном времени
journalctl -u content-bot -f
```

---

## 🚀 Удобный скрипт обновления (рекомендуется)

```bash
cd /home/content-bot && ./deploy.sh
```

**Этот скрипт:**

* Скачивает последние изменения с GitHub
* Обновляет зависимости
* Перезапускает бота

---

## 🤖 Как пользоваться ботом (для пользователей)

1. Найдите бота: `@poly_content_bot`
2. Напишите `/start`
3. Отправьте фото + короткую историю/подпись
4. Добавьте теги (по желанию)
5. Бот сохранит материал автоматически

## 🛠 Как вносить изменения в код (для разработчиков)

### Порядок работы:

```bash
# Создание новой ветки
git checkout -b feature/название-задачи
```

После завершения:

1. Запушьте изменения в GitHub
2. На сервере выполните:

```bash
cd /home/content-bot && ./deploy.sh
```

> ⚠️ **Важно:**
> Не редактируйте файлы напрямую на сервере (кроме `.env`).
> Все изменения должны идти через Git.

## 🔧 Полезные команды

| Команда                         | Описание                       |
| ------------------------------- | ------------------------------ |
| `./deploy.sh`                   | Полное обновление + перезапуск |
| `systemctl status content-bot`  | Статус бота                    |
| `journalctl -u content-bot -f`  | Логи в реальном времени        |
| `systemctl restart content-bot` | Перезапустить бота             |

## ⚠️ Важные правила

* Никогда не коммитьте файл `.env`
* После каждого `git push` в `main` — запускайте `./deploy.sh`
* Если бот не отвечает — сначала посмотрите логи:

```bash
journalctl -u content-bot -f
```
