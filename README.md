# 📸 UGC Telegram Bot — Poly Content Bot

## 🎯 Цель проекта

Создать полностью рабочего Telegram-бота, который собирает фотографии и короткие истории от студентов и сотрудников университета.
Бот автоматически сохраняет контент на **Яндекс.Диск** + в **Google Sheets** для дальнейшего использования в соцсетях, подкастах и презентациях.

---


## 🚀 Как начать работу (для каждого участника)

### 1. Локальная разработка

```bash
# Клонируем репозиторий
git clone https://github.com/YatchenkoArD/-UGC-Bot-Telegram.git
cd -UGC-Bot-Telegram

# Создаём виртуальное окружение
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate

# Устанавливаем зависимости
pip install -r requirements.txt

# Создаём .env файл
cp .env.example .env
# ← Открой .env и вставь свой BOT_TOKEN
```

### ▶️ Запуск бота локально

```bash
python main.py
```

# 📌 План распределения задач по проекту Telegram-бота

## 🎯 Цель проекта
Создать полностью рабочего Telegram-бота, который:
- собирает фотографии и короткие истории
- сохраняет данные на Яндекс.Диск
- записывает информацию в Google Sheets
- подготавливает данные для соцсетей, подкастов и презентаций

---

## 🔧 1. Backend-логика бота (ядро)

**Задачи:**
- обработка сообщений (фото + текст)
- валидация данных (длина истории, формат фото)
- реализация сценария диалога
- обработка ошибок (некорректный ввод)

---

## ☁️ 2. Интеграция с хранилищами

**Задачи:**
- загрузка фото на Яндекс.Диск
- запись данных в Google Sheets:
  - имя (если есть)
  - текст истории
  - ссылка на фото
  - дата
- обработка сбоев API
- retry-логика
- логирование

---

## 🧠 3. Архитектура данных

**Задачи:**
- проектирование структуры Google Sheets:
  - колонки
  - формат
- единый формат хранения данных
- система именования файлов на диске

---

## 🎨 4. UX / сценарий взаимодействия

**Задачи:**
- тексты сообщений бота (tone of voice)
- пользовательский сценарий:
  1. приветствие
  2. инструкция
  3. загрузка фото
  4. ввод истории
  5. подтверждение
- кнопки и меню

---

## 🧪 5. Тестирование и QA

**Задачи:**
- написание тест-кейсов:
  - нет фото
  - слишком длинный текст
  - спам
- нагрузочное тестирование
- проверка:
  - загрузки на диск
  - записи в таблицу

---

## 📊 6. Контент и модерация (опционально)

**Задачи:**
- проверка историй
- фильтрация контента
- добавление статусов:
  - "новое"
  - "одобрено"
  - "в работу"
